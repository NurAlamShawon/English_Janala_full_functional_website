Frequently Asked Questions

1.The difference between var, let, and const?
In JavaScript, var, let, and const are used to declare variables, but they behave differently. The var keyword is the oldest way to declare a variable. It is function-scoped, which means the variable is accessible anywhere inside the function where it is declared. Variables declared with var can also be re-declared and updated. However, it can cause unexpected results in modern code, so it's less recommended now. The let keyword was introduced in ES6. It is block-scoped, meaning it only works within the curly braces {} where it is declared, like inside an if statement or a loop. Variables declared with let can be updated, but cannot be re-declared in the same scope. It is a better and safer choice than var for most situations. The const keyword is also block-scoped like let, but it is used to declare constants. Once a variable is declared with const, its value cannot be changed or updated. It is useful when you want to make sure a variable never changes.
2.The difference between map(), forEach(), and filter()?
In JavaScript, map(), forEach(), and filter() are all methods used to work with arrays, but they serve different purposes. The forEach() method is used when you want to go through each item in an array and perform an action, such as printing the values or updating something outside the array. It does not return a new array, and it’s mostly used when you don’t need to save the result, just do something with the items. The map() method is used when you want to create a new array by applying some changes to each item in the original array. It takes each element, processes it, and returns a new array with the transformed values. This is helpful when you want to modify data while keeping the original array unchanged. On the other hand, the filter() method is used when you want to remove or keep certain elements based on a condition. It goes through each item in the array and returns a new array with only the items that match the condition you give it.
3.Explain arrow functions and how they are different from regular functions?
In JavaScript, arrow functions are a shorter way to write functions. They were introduced in ES6 and are commonly used because they make the code cleaner and easier to read. The main difference is in the syntax. A regular function is written using the function keyword, while an arrow function uses the => symbol. Besides the syntax, arrow functions behave differently in terms of how they handle the this keyword. In regular functions, the value of this depends on how the function is called. But in arrow functions, this is inherited from the surrounding scope, meaning it doesn't change. Because of this behavior, arrow functions are very useful in situations like event listeners or callbacks where we want to keep the same this context. However, arrow functions cannot be used as constructors, and they don’t have their own arguments object like regular functions do.
4.How JavaScript Promises work?
In JavaScript, a Promise is a way to handle tasks that take some time to complete, like loading data from a server or waiting for a response from an API. These tasks are called asynchronous operations. A Promise does not give the result immediately. Instead, it acts like a container that will eventually give us either a successful result or an error. Every Promise has three states. First, it starts as pending, which means the task is still running. If the task completes successfully, the Promise becomes fulfilled, and we get the result. But if the task fails, the Promise becomes rejected, and we get an error message. We can create a Promise using the new Promise() syntax, and inside it, we define what should happen when the task succeeds or fails using two functions: resolve for success and reject for failure. To handle the result of a Promise, we use .then() to run code when the Promise is successful, and .catch() to handle errors if something goes wrong.
5.How closures work in JavaScript?
In JavaScript, a closure is created when a function is defined inside another function and the inner function has access to the outer function’s variables, even after the outer function has finished running. This means that the inner function "remembers" the environment in which it was created. It can still access the outer function’s variables because of how JavaScript manages scope and memory. For example, if we declare a variable inside a function, and then return another function from it, that inner function keeps a reference to the outer variable, not just a copy of its value. This allows us to preserve data in memory without using global variables, which is useful for things like private variables or function factories.
